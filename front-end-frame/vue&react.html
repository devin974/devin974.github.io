    Vue 与 React 的不同理解心得

    devin 11/20/2021 框架vuereact

    # 前言

    之前工作的主要技术都是以Vue为主，后也因工作原因需要接触使用React，所以最近也把之前写过demo的一些经验捡起来，再深入去了解使用，在这篇文章中会记录我所感受和理解的 React 和 Vue之间的区别。

    # Vue 和 React 的共同点

    • 都是用于建立 UI 的 JavaScript 库。
    • 都使用 Virtual Dom(虚拟DOM)。
    • 都基于组件化、单文件组件规范。
    • 都是单向数据流。
    • 都有路由与状态管理库。

    其实上述几点列出来，也是最近的学习感受，本质来说两个框架都是一样的，只是差别在于两个框架在实现时候的设计思想。

    在写 Vue 的时候，每个组件就像平常开发一样，由 HTML + JavaScript + CSS 组成。 在写 React 的时候，其实很大程度上会感觉自己就是在写 JavaScript 一样。

    # Vue 与 React 的核心思想的不同

    Vue 主打灵活易用的渐进式 JavaScript 框架，也就是希望降低前端开发的门槛，因此很多人说 Vue 是目前三大框架中学习曲线最平缓的，而 Vue 的作者尤雨溪也曾这样回答：

    「Vue 使用的是 web 开发者更熟悉的模板与特性，Vue 的 API 跟传统 web 开发者熟悉的模板契合度更高，比如 Vue 的单文件组件是以模板(template) + JavaScript + CSS 的组合模式呈现，它跟 web 现有的 HTML、JavaScript、CSS 能够更好地配合。React 的特色在于函数式编程的理念和丰富的技术选型。Vue 比起 React 更容易被前端工程师接受，这是一个直观的感受；React 则更容易吸引在 FP 上持续走下去的开发者。」

    Vue 原先的定位是尽可能的降低前端开发的门槛，也因此可以从这个出发点理解为什么 Vue 推崇所谓的 渐进式开发体验，对于一个初学的前端开发而言，数据可变动、数据双向绑定，每个 .vue 单文件都包含 html(template)、javascript、css 等，其实与既有的所谓 经典web开发方式 所差无几，也就相对于比较好理解与学习，这个核心价值的开展也或许可以归结到尤雨溪本身是自由开发者。

    React 早期的口号是 Rethinking Best Practices。 明确的表示要用更好的方式去颠覆前端开发（在 jQuery 称霸时空背景下，现在看来确实完美颠覆了）。而在 React 中推崇函数式编程（FP），其中包含数据不可变及单向数据流等，在 Pure Function 的原则下，最大优点当然是所谓的稳定性及可测试/可维护性（同样的输入，必得到同样的输出），而这也是为什么 React 更适合大型、复杂应用的原因。

    在写 Vue 时，会有 template、css、script 三个区域组合，就像我们最开始在学习写网页时的 html、css、js 一样。 在 Vue 的官方文件中，也提到如此的设计思想是为了拥抱经典的 web 开发技术。

    # 状态数据绑定的不同

    # Vue 双向绑定 vs React 单向绑定

    或许这也可以说是两者间最大的区别，Vue 提供响应式的数据，也就是当我们使用 v-model 将数据劫持时，只要数据有所改变，界面就会自动更新，而在 React 中，我们则需要通过 onChange 或 setState 方法来完成这件事。Vue 的作者尤雨溪将这两者说明为 Push-based 和 Pull-based。

    # Push-based：Vue

    改变数据后，数据本身会把这个改动发布出去，告诉页面进行渲染。

    # Pull-based：React

    我们需要额外的主动去通知系统你该进行渲染，这时系统才会帮我们渲染页面。

    # 监听数据变化与渲染过程的不同

    # Vue

    • 依赖收集，自动优化，可变。
    • 监听 data 的所有属性，可以直接修改。
    • 当数据改变时，自动找到引用组件重新渲染。

    # React

    • 基于数据状态，手动优化，不可变。
    • 需要通过 onChange 或 setState 等方法进行修改
    • 以组件为根目录，预设为全部重新渲染。

    在 React 当某个组件的状态发生变化的时候，会以该组件为根，重新渲染整个子组件，因此在写 React 时，我们常会思考是不是会造成不必要的子组件的渲染，因此就需要在所有可能的地方使用PureComponent，或是手动实现 shouldComponentUpdate 方法，但如此一来，我们也就必须保证该组件的的整个子组件渲染输出的影响都是由该组件的 props所决定，不然就可能导致例外且难以觉察的渲染结果不一致，导致在组件优化的考量上，会耗费比较多的心力。

    但在 Vue 中，因为组件的依赖引用在渲染过程中是自动追踪的，所以系统精确知道哪些组件需要被重新渲染，我们可以简单理解为在 Vue 中的每个组件都自动获得 shouldComponentUpdate ，因此在 Vue 开发而言，这个特点让开发者相较而言不需要考虑这一类的性能优化问题，而能更加专注于开发。

    更清楚的说 Vue 之所以可以更快的计算出 Virtual DOM 的差异，是因为它在渲染过程中，会追踪每一个的依赖关系（通过 getter/setter 等），不需要重新渲染整个dom树，而 React 则是通过引用比较的方式(diff)，因此如果开发不注意可能会导致大量且不必要的 VDOM 重新渲染，说到这里可能会好奇？

    这就要回归到最开始的两个框架的设计理念区别，Vue 资料的「可变」，和 React 资料的「不可变」，而这两者其实没有真实所谓的优劣之分，Vue 在这点上可以说是开发更简单，但 React 则是在构建大型应用的时候，能更好的维护。

    # 生命周期

    综上所述，Vue 中的组件也与 React 一样，有所谓类似生命周期的方法，例如当组件就绪，挂载到页面之前 (created)，挂载到页面时(mounted)、状态更新时(updated)、组件销毁时(unmounted)等 hook，而两者最大的差别在于先前说的，Vue 本身是具有响应式的，也就是已经预设了 shouldComponentUpdate ，所以在 Vue 里当然就不需要它。

    # diff 的算法不同

    Vue 与 React 在实践 Virtual Dom 的方法是相当类似的，都是基于以下两点，将传统 Diff 算法从 O(n³) 降为 O(n)。

    • 不同的组件产生不同的 DOM 结构。
    • 当 type 不相同时，对应 DOM 操作就是直接删除旧的 DOM ，创建新的 DOM。
    • 同一层级的子节点通过唯一的key来作区分。

    # React 的 Diff 算法核心实现

    1. 首先对新集合进行遍历，for( name in nextChildren)
    2. 通过唯一 key 来判断老集合中是否存在相同的节点。如果没有的话就创建
    3. 如果有的话，if (preChild === nextChild )
    4. 在新集合中的位置和在旧集合中 lastIndex 进行比较
    5. 如果if (child._mountIndex < lastIndex) 进行移动操作，否则不进行移动操作
    6. 如果遍历的过程中发现在新集合中没有，但在老集合中有的节点，会进行删除操作

    # Vue 的 Diff 算法核心实现

    updateChildren 是 vue diff 的核心，过程可以概括为：

    旧 children 和新 children 各有两个首尾的变量 StartIndex 和 EndIndex，他们的 2 个变量相互比较，一共有 4 种比较方式。

    如果 4 种比较都没有匹配，如果设置了 key， 就会用 key 进行比较，在比较的过程中，变量会往中间靠，一旦StartIndex > EndIndex 表示旧 children 和新 children 至少有一个已经遍历完了，就会结束对比。

    Vue 是基于 snabbdom 库，采用两端比较的 diff 算法，同事从新旧 children 的两端开始进行比较，借助 key 值找到可复用的节点，再进行相关操作。也就是说Vue Diff是采用边对比边更新，而React则是使用 diff 队列保存需要更新那些 DOM ，算出 patch 树后，在统一操作批量更新 DOM，相比较下， Vue 可以减少移动节点次数，减少不必要的性能损耗，更加的优雅。

    # 数据流向与传递不同 Event vs Callback

    # Vue

    • 父组件通过 props 向子组件传递数据或者 callback，但一般只传递数据。
    • 子组件通过自定义事件(custom event)向父组件传递数据。（实现方式类似于父层监听子层的事件变化）
    • 以 provide/inject 来实现父组件向子组件跨多级的数据传递。

    # React

    • 与 Vue 一样，父组件通过 props 向子组件传递数据或者 callback。
    • 通过 context 进行跨层级的数据传递，和 provide/inject 作用差不多。
    • React 本身并不支持自定义事件。

    Vue 与 React 都有方式可以传递达到子级向父级传递数据，但 Vue 倾向于自定义事件(custom event)传递，而 React 则是使用回调函数。

      //子
  export default {
    data() {
      ...
    },
    props: ["delay"],
    methods: {
      stopTimer() {
        //定义 event名称,和放入要传递的参数
        this.$emit("end", this.reactionTime); 
      },
    },
  };

  //父
  <template>
    <Block v-if="isPlaying" :delay="delay" @end="stopGame" /> 
    //触发custom event时，会触发父级method
    <div v-if="showResult">spent {{ reactionTime }}</div>
  </template>

<script>
  import Block from "./components/Block.vue";

  export default {
    name: "App",
    components: { Block },
    data() {
      ...
    },
    methods: {
      stopGame(time) { 
        // 通过event emit定义的custom event
        // 在触发时会自动传入所传递的参数
        this.reactionTime = time;
        this.isPlaying = false;
        this.showResult = true;
      },
    },
  };
</script>

    # 事件机制的不同

    # Vue

    • 原生事件使用标准web事件，（例CustomEvent 也是其中一种）。
    • 具有自定义事件(custom event)，让父组件可以实现对子组件的监听。
    • 利用 snabbdom 库的模块插件。

    # React

    原生事件经过包装，所有事件会先冒泡到顶层 document 监听后，才合成事件向下传递，因此可以跨端使用事件机制，而不是经过 web dom 的强绑定。父子间都是通过 props 做为数据传递，并无向上传递。

    # 框架本质的不同

    Vue 本质是 MVVM 框架（渐进式框架），由 MVC 发展而来； React 是前端组件化框架，由后端组件化发展而来。

    # 个人感受小结

    # Vue 的优势

    • 更快的渲染速度和更小的体积
    • 简单的语法及项目创建，尤其是通过 vue cli 可以依项目快速选择相应的包/功能模块，在创建时更为便利且快速。
    • 响应式数据状态，使开发更加便利

    # React 的优势

    • 适用于 Web 端和原生 App
    • 适用于大型应用和更好的可测试/维护性
    • 更大的生态圈带来的更多支援和工具

    其实当时在学习 react的时候， 一开始确实花了一些时间理解，但其实并没有感受到太大的阻碍，可能很大一部分原因跟我个人很喜欢 JS 有关，再加上是以 React 的 Function Component 写法开发，理解基本的 hook 如何使用后，就算不看文档也可以順利开发，不会有太大的问题。

    但学习 Vue ，因为有各种各样的 api 操作，虽然上手简单，但要能够熟练以至于提高开发效率，过程需要不断翻看文档 明白各项配置才能真的上手。而相较于此，React 沒有所谓这些林林总总的 api，因此学习起来，反而感觉 React 像是一张可以自由挥洒的白纸，而 Vue 则像是积木，你必须知道各种 api 的使用方式，才可以将整个页面构建出来。
