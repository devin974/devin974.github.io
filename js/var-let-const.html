    var、let 、const详解

    devin 5/29/2021 JavaScriptes6

    前言：
    首先，我们先要搞懂这两个前置问题，ECMAScript 和 JavaScript 到底是什么关系？什么是es5?什么是es6?

    # ECMAScript 和 JavaScript 到底是什么关系？

    1. ECMAScript是什么？

    ECMAScript是一种由Ecma国际（前身为欧洲计算机制造商协会，European Computer Manufacturers Association）通过ECMA-262标准化的脚本程序设计语言。这种语言在万维网上应用广泛，它往往被称为JavaScript或JScript，所以它可以理解为是JavaScript的一个标准,但实际上后两者是ECMA-262标准的实现和扩展。

    1. JavaScript是什么？

    JavaScript一种直译式脚本语言，是一种动态类型、弱类型、基于原型的语言，内置支持类型。它的解释器被称为JavaScript引擎，为浏览器的一部分，广泛用于客户端的脚本语言，最早是在HTML（标准通用标记语言下的一个应用）网页上使用，用来给HTML网页增加动态功能。

    1. ECMAScript 和 JavaScript 有什么关系？

    JavaScript是通用的跨平台脚本语言，遵守ECMA-262标准（ES标准），可以理解为是ECMAScript的方言。为了取得技术优势，微软推出了JScript，CEnvi推出ScriptEase，与JavaScript同样可在浏览器上运行。所以为了统一规格，因为JavaScript兼容于ECMA标准，因此也称为ECMAScript。

    # 什么是es5?什么是es6?

    1. es5：
      ECMAScript 5.1 (以下简称 ES5) 是ECMAScript(基于JavaScript的规范)标准最新修正。 与HTML5规范进程本质类似，ES5通过对现有JavaScript方法添加语句和原生ECMAScript对象做合并实现标准化。ES5还引入了一个语法的严格变种，被称为”严格模式(strict mode)”。
    2. es6：
      ECMAScript 6.0（以下简称 ES6）是 JavaScript 语言的下一代标准，已经在 2015 年 6 月正式发布了。

    # var、let、const是什么？

    var、let、const 都是javascript中声明变量的方式。

    # var、let、const有什么用？

    都可以声明变量，只是声明的方式和相关特性不一样。

    # var、let、const有什么区别？

    # var（es5）

    • 特性
      • var 变量声明提升
      • 可重复声明
      • 无块级作用域的概念

    # let（es6新增）

    • 特性
      • 无变量声明提升，必须先声明后使用
      • 不能重复声明相同变量
      • 有块级作用域的概念
      • 暂时性死区

    # const（es6新增）

    • 特性
      • 无变量声明提升，必须先声明后使用
      • 不能重复声明相同变量
      • 声明的变量为常量不允许更改，使用时必须初始化（赋值）（引用数据类型的属性值可更改）
      • 有块级作用域的概念
      • 暂时性死区

    # let、const解决了什么问题？

    # 变量重复声明的隐患

    # 块级作用域问题

    ES5只有全局作用域和函数作用域，没有块级作用域。

    由此带来了一些问题：

    1. 内层变量可能覆盖外层变量；
    2. 用来计数的循环变量泄露为全局变量
    var tmp = new Date();
function f() {
  console.log(tmp); // 想打印外层的时间作用域，被if内部tmp影响
  if (false) {
    var tmp = 'hello world'; // 这里声明的作用域为整个函数，tmp会被提升到f函数内的顶部
  }
}
f(); // undefined

var s = 'hello';
for (var i = 0; i < s.length; i++) {
  console.log(s[i]); // i 理想应该是 0 1 2 3 4，实际得到的是5 5 5 5 5
}
console.log(i); // 5 全局范围都可以读到

function test(){
    if(false){
        var name1 = "小红";
        let name2 = "小明";
        const name3 = "小张";                
    }else{
        console.log(name1) // undefined
        console.log(name2) // name2 is not defined
        console.log(name3) // name3 is not defined
    }
}

    # 暂时性死区问题

    暂时性死区和不能变量声明提升主要是为了减少运行时错误，防止在变量声明前就使用这个变量，从而导致意料之外的行为。

    # let、const使用场景

    1. let使用场景：声明可变变量，用以替代var。
    2. const使用场景：常量、声明匿名函数、箭头函数等不变的数据的时候。
